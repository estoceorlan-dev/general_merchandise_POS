# JCE POS

Offline-first Flutter scaffold for a multi-branch General Merchandise Point of
Sale system.

## Platforms

- Android
- Windows
- Web
- Linux scaffold generated by Flutter and available for future use

## Architecture

The project uses feature-first folders under `lib/features`, shared models and
providers under `lib/shared`, and app-wide infrastructure under `lib/core`.

Key foundations:

- Riverpod for dependency injection and state access
- GoRouter for centralized route management
- Drift and SQLite package setup, with schema work intentionally deferred
- Firebase package setup with an initialization service ready for FlutterFire
  options
- Repository contracts for auth, POS sales, products, inventory, transfers,
  purchases, reports, logs, users, and settings
- Light and dark themes with shared colors, spacing, radii, and typography
- Responsive desktop sidebar and mobile bottom navigation

## Verification

```sh
flutter analyze
flutter test
```

## Firebase

Firebase packages are installed, but no project-specific options are committed.
Run FlutterFire configuration before calling `FirebaseInitializationService` in
production startup code.

## Local Database

Drift dependencies and database configuration scaffolding are present in
`lib/core/database`. No tables or generated database files are included yet.
